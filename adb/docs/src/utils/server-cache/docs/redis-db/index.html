REDIS Server

Redis = Remote Dictionary Server

  • API

Install (Ubuntu)â€‹

sudo apt update

sudo apt install redis-server

Configâ€‹

Basic configâ€‹

/etc/redis/redis.conf

# bind to localhost or ip
bind 127.0.0.1

# set below to manage Redis as a service
supervised systemd

# set very strong password 
requirepass Thl9RrV6mi550dqMmiaxRKiMPlTbQGO4yOshtYhqF9

# change default port 6379
port 7480

# loglevel =  debug, verbose, notice, warning
loglevel notice

# logfile  (file must have permissions)
logfile /var/log/redis/redis-server.log

# db directory location
dir /var/lib/redis

# run in background
daemonize yes

# pid file (file must have permissions)
pidfile /var/run/redis/redis-server.pid

# maxclients 10000

disable dangerous commandsâ€‹

  • Few dangerous commands are FLUSHDB, FLUSHALL, KEYS, PEXPIRE, DEL, CONFIG, SHUTDOWN, BGREWRITEAOF, BGSAVE, SAVE, SPOP, SREM, RENAME, and DEBUG

  • Either rename them or disable by setting blank

/etc/redis/redis.conf

rename-command FLUSHDB ""
rename-command FLUSHALL ""
rename-command DEBUG ""
# rename to custom name
rename-command CONFIG CONFIG_CUSTOM

backup settings (RDB)â€‹

  • RDB (Redis Database Backup)
    • point in time snapshot at specified interval & key changes
    • Restores automatically if dump file exists on restart of service

/etc/redis/redis.conf

# The filename where to dump the DB
dbfilename dump.rdb

# Save the DB on disk:
#   save <seconds> <changes>

#   In the example below the behaviour will be to save:
#   after 900 sec (15 min) if at least 1 key changed
#   after 300 sec (5 min) if at least 10 keys changed
#   after 60 sec if at least 10000 keys changed

save 900 1
save 300 10
save 60 10000

backup settings (AOF)â€‹

  • Append only file (AOF) file
    • Keeping log of write operations performed by redis

/etc/redis/redis.conf

# yes/no
appendonly yes

appendfilename "appendonly.aof"

# no: don't fsync, just let the OS flush the data when it wants. Faster.
# always: fsync after every write to the append only log. Slow, Safest.
# everysec: fsync only one time every second. Compromise.
appendfsync always

  • aof file will be priority on startup if both aof & rdb exists as its has better durability.

Service commandsâ€‹

sudo systemctl start redis

redis-server --version

Noteâ€‹

RDB backupâ€‹

  • Advantages

    • compact, single file of redis data
    • perfect for backups
    • performance is good
    • allows faster restarts

  • Disadvantages

    • Chances of data loss
    • Fork process may impact write operations for few milliseconds

AOF backupâ€‹

  • Advantages

    • High durability
    • Allows different fsync policies
    • Append only logs, no chance of corruption
      • Corrupted file can be fixed by redis-check-aof tool
    • AOF file can be parsed and lines can be removed

  • Disadvantages

    • Size is bigger than rdb
    • Slower than rdb
