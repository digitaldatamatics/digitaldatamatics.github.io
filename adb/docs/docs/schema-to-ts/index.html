Schema to TS

  • Converting JSON schema to typescript

Install dependenciesâ€‹

npm install --save-dev glob cpy chokidar esm
npm install --save-dev json-schema-to-typescript
npm install --save-dev @types/json-schema # for types like JSONSchema4 used in mongodb validations

Code filesâ€‹

  • build-scripts/schema.js
  • build-scripts/watch-schema.js

Package scriptâ€‹

  • Following scripts are added in package.json
{
    "scripts":{
       "schema": "node -r esm build-scripts/schema.js",
       "watch-schema": "node -r esm build-scripts/watch-schema.js",
    }
}

Usageâ€‹

# to convert all schemas to typescript types
npm run schema

# to observe for schema changes runtime & convert to typescript types (development mode)
npm run watch-schema

Othersâ€‹

enumsâ€‹

  • https://github.com/bcherny/json-schema-to-typescript/issues/200
  • https://github.com/bcherny/json-schema-to-typescript/pull/405
//input
{
    //values
    "enum": [
            "pass",
            "fail"
        ],
     //enum variable name   
    "title": "GRADE_STATUSES",

    //keys
    "tsEnumNames": [
        "PASS",
        "FAIL"
    ],
}

//output
export const enum GRADE_STATUSES {
  PASS = "pass",
  FAIL = "fail"
}

Relative paths in JSON schemaâ€‹

  • by default paths must be relative from current working directory
    • Internal lib docs
//example
{
 "$ref": "src/models/json-schemas/common-schema.json"
}
  • but for files in "src/models/json-schemas/", cwd (current working directory) is changed to "src/models/json-schemas" in "build-scripts/schema.js" to use relative paths accordingly.
    • Why ? To support VS code intellisense for json-samples folder 
{
    "$ref": "./common-schema.json"
}
